# CompTIA A+ 220-1201 V15 Objectives

About the Exam Candidates are encouraged to use this document to help prepare
for the CompTIA A+ 220-1201 certification exam In order to receive the CompTIA
A+ certification, you must pass two exams: Core 1 (220-1201) and Core 2
(220-1202) The CompTIA A+ Core 1 (220-1201) and Core 2 (220-1202) certification
exams will verify the successful candidate has the knowledge and skills required
to:

Install, configure, and maintain computer equipment, mobile devices, and
software for end users Service components based on customer requirements
Understand networking basics and apply basic cybersecurity methods to mitigate
threats Properly and safely diagnose, resolve, and document common hardware and
software issues Apply troubleshooting skills and provide customer support using
appropriate communication skills Understand the basics of scripting, cloud
technologies, virtualization, and multi-OS deployments in corporate environments

TROUBLESHOOTING METHODOLOGY KNOWLEDGE During the job task analysis workshop for
the A+ 220-1200 series, subject matter experts deemed the troubleshooting
methodology an effective best practice that new job incumbents should be aware
of and leverage as they engage in troubleshooting new issues on the job.
However, while this methodology is practical, the decision was made to not
include it in the exam. While the methodology itself will not be tested, there
remains an emphasis on troubleshooting within the job role context. Therefore,
the troubleshooting methodology section appears here as part of this “competency
standard” but does not constitute a formal objective or part of the A+
certification exam Training institutions that prepare individuals with very
little technical knowledge and experience are encouraged to leverage this
methodology, especially when such individuals might be applying for their first
IT job The troubleshooting methodology includes the following steps:

- Identify the problem
- Establish a theory of probable cause (question the obvious)
- Research knowledge base/internet, if applicable
- Test the theory to determine the cause
- Establish a plan of action to resolve the problem and implement the solution
- Verify full system functionality and, if applicable, implement preventive
  measures
- Document findings/lessons learned, actions, and outcomes

## 2.0 Networking 23%

### 2.1 Compare and contrast Transmission Control Protocol (TCP) and User Datagram Protocol (UDP) ports, protocols, and their purposes

- **{{Transport Layer}}**: Layer responsible for *end-to-end communication* and data flow.
- **{{Transmission Control Protocol (TCP)}}**: *Connection-oriented* transport providing reliable data delivery.
  - **{{Three-Way Handshake}}**: Process to *establish a reliable* connection (SYN, SYN-ACK, ACK).
  - **{{Flow Control}}**: Managing data rate to *prevent overwhelming* the receiver.
  - **{{Sequencing}}**: Reordering segments into *the correct original order*.
  - **{{Acknowledgment (ACK)}}**: *Confirmation of receipt* for transmitted data segments.
- **{{User Datagram Protocol (UDP)}}**: *Connectionless* transport providing fast, best-effort delivery.
  - **{{Best-Effort Delivery}}**: Transmitting data *without guaranteeing arrival* or order.
- **{{Port}}**: Logical identifier for *specific processes or services*.
- **Ports and protocols**
  - **{{File Transfer Protocol (FTP)}}**: Protocol for *transferring files* between systems (TCP 20, 21).
  - **{{Secure Shell (SSH)}}**: *Encrypted* remote console and file transfer (TCP 22).
  - **{{Telnet}}**: *Unencrypted* remote console access (TCP 23).
  - **{{Simple Mail Transfer Protocol (SMTP)}}**: *Sending* email between servers (TCP 25).
  - **{{Domain Name System (DNS)}}**: *Resolves* domain names to IP addresses (UDP/TCP 53).
  - **{{Dynamic Host Configuration Protocol (DHCP)}}**: *Dynamically assigns* IP addresses to hosts (UDP 67, 68).
  - **{{Hypertext Transfer Protocol (HTTP)}}**: *Unencrypted* web traffic communication (TCP 80).
  - **{{Post Office Protocol 3 (POP3)}}**: *Retrieves* email and typically deletes from server (TCP 110).
  - **{{Internet Mail Access Protocol (IMAP)}}**: *Retrieves* email while syncing with server (TCP 143).
  - **{{NetBIOS / NetBT}}**: *Legacy* naming and session services (UDP 137, 138; TCP 139).
  - **{{Lightweight Directory Access Protocol (LDAP)}}**: *Accessing and maintaining* distributed directory information (TCP 389).
  - **{{Hypertext Transfer Protocol Secure (HTTPS)}}**: *Encrypted* web traffic communication (TCP 443).
  - **{{Server Message Block (SMB) / CIFS)}}**: *File and printer sharing* in Windows (TCP 445).
  - **{{Remote Desktop Protocol (RDP)}}**: *Graphical* remote desktop access (TCP 3389).
- **{{TCP vs. UDP}}**: *Reliability and overhead* vs speed and efficiency.

### 2.2 Explain wireless networking technologies

- **{{Wireless Networking}}**: Communication using *radio waves instead of wires*.
- **Frequencies**
  - **{{2.4 GHz}}**: *Longer range* with more interference and congestion.
  - **{{5 GHz}}**: *Faster speeds* with shorter range and less interference.
  - **{{6 GHz}}**: *Latest frequency* providing highest speeds and least congestion.
- **Channels**
  - **{{Regulations}}**: Legal *limits on radio frequency* usage and power.
  - **{{Channel Selection}}**: Choosing *optimal frequencies* to avoid interference.
    - **{{Widths}}**: *Size of frequency band* (e.g., 20 MHz, 40 MHz).
    - **{{Bands}}**: Specific *ranges of frequencies* assigned for use.
- **{{Bluetooth}}**: Short-range wireless for *personal area network (PAN)* device pairing.
- **{{802.11 Standards}}**: IEEE family of *specifications for wireless local area networks*.
  - **{{802.11a}}**: Legacy 5 GHz standard *supporting up to 54 Mbps*.
  - **{{802.11b}}**: Legacy 2.4 GHz standard *supporting up to 11 Mbps*.
  - **{{802.11g}}**: Legacy 2.4 GHz standard *supporting up to 54 Mbps*.
  - **{{802.11n (Wi-Fi 4)}}**: Supports 2.4/5 GHz *using MIMO for increased throughput*.
  - **{{802.11ac (Wi-Fi 5)}}**: 5 GHz standard *using MU-MIMO for higher efficiency*.
  - **{{802.11ax (Wi-Fi 6)}}**: High-efficiency standard *supporting 2.4, 5, and 6 GHz*.
- **{{Near-Field Communication (NFC)}}**: Extremely short-range *contactless data exchange and payments*.
- **{{Radio-Frequency Identification (RFID)}}**: Uses electromagnetic fields to *identify and track tags automatically*.
- **{{Service Set Identifier (SSID)}}**: The *human-readable name* of a wireless network.
- **{{MIMO / MU-MIMO}}**: Using *multiple antennas* to increase data throughput.
- **{{CSMA / CA}}**: Protocol *preventing data collisions* in wireless networks.

### 2.3 Summarize services provided by networked hosts

- **Server roles**
  - **{{DNS Server}}**: Host responsible for *resolving domain names* to IP addresses.
  - **{{DHCP Server}}**: Host automatically *assigning IP configurations* to clients.
  - **{{File Share / File Server}}**: Central storage for *storing and sharing* documents.
  - **{{Print Server}}**: Manages *print requests* and queue for network printers.
  - **{{Mail Server}}**: System for *sending, receiving, and storing* email.
  - **{{Syslog Server}}**: Centralized repository for *collecting and storing* system logs.
  - **{{Web Server}}**: Hosts websites and *serves content via HTTP/HTTPS*.
  - **{{Authentication, Authorization, and Accounting (AAA)}}**: Framework for *controlling access and tracking* usage.
  - **{{Database Server}}**: System for *storing and managing structured* data.
  - **{{Network Time Protocol (NTP) Server}}**: Provides *time synchronization* for network devices.
- **Internet appliances**
  - **{{Spam Gateway}}**: Appliance filtering *unwanted or malicious* email traffic.
  - **{{Unified Threat Management (UTM)}}**: All-in-one appliance combining *multiple security functions*.
  - **{{Load Balancer}}**: Distributes network traffic across *multiple servers* for efficiency.
  - **{{Proxy Server}}**: Intermediary acting on *behalf of clients* for requests.
- **Legacy / embedded systems**
  - **{{Supervisory Control and Data Acquisition (SCADA)}}**: Systems used to *control industrial processes* and infrastructure.
  - **{{Embedded System}}**: Computer system *dedicated to specific function* within device.
- **{{Internet of Things (IoT) Devices}}**: Network of physical objects *embedded with sensors/software*.
  - **{{Smart Devices}}**: Everyday objects with *computing capability and connectivity*.

### 2.4 Explain common network configuration concepts

- **{{Domain Name System (DNS)}}**: Hierarchical system for *resolving hostnames to IP addresses*.
  - **{{A Record}}**: Maps a hostname to *an IPv4 address*.
  - **{{AAAA Record}}**: Maps a hostname to *an IPv6 address*.
  - **{{Canonical Name (CNAME)}}**: Creates *an alias for existing* domain records.
  - **{{Mail Exchanger (MX)}}**: Identifies *mail servers responsible* for a domain.
  - **{{Text (TXT)}}**: Stores *arbitrary text information* for domain verification.
    - **Spam management**
      - **{{DomainKeys Identified Mail (DKIM)}}**: Cryptographic *signature verifying email source* authenticity.
      - **{{Sender Policy Framework (SPF)}}**: List of *authorized hosts permitted* to send mail.
- **{{Domain-based Message Authentication, Reporting, and Conformance (DMARC)}}**: Framework *using SPF and DKIM* to prevent spoofing.
- **{{Dynamic Host Configuration Protocol (DHCP)}}**: Automatically *assigns IP addresses and settings* to hosts.
  - **{{DORA Process}}**: Four-step sequence *to obtain IP configuration* (Discover, Offer, Request, Acknowledge).
  - **{{Lease}}**: The *duration a client* holds an IP address.
  - **{{Reservation}}**: Permanent *IP assignment based* on MAC address.
  - **{{Scope}}**: Range of *available IP addresses* for assignment.
  - **{{Exclusion}}**: Specific *addresses removed from* the assignment pool.
- **{{Virtual LAN (VLAN)}}**: Logical *segmentation of a physical network* at Layer 2.
  - **{{Tagging (802.1Q)}}**: Adding *ID headers to frames* for identification.
- **{{Virtual Private Network (VPN)}}**: Encrypted *tunnel providing secure remote access* over internet.
  - **{{Tunneling}}**: Encapsulating *data packets within another* protocol.

### 2.5 Compare and contrast common networking hardware devices

- **{{Router}}**: Forwards data packets *between different computer networks*.
- **{{Switch}}**: Connects devices within *a single local network*.
  - **{{Managed Switch}}**: Allows *configuration and monitoring* of ports.
  - **{{Unmanaged Switch}}**: Fixed configuration *providing basic connectivity* only.
- **{{Access Point (AP)}}**: Extends *wireless connectivity to a* wired network.
- **{{Patch Panel}}**: Static panel *organizing and connecting* network cables.
- **{{Firewall}}**: Filters network traffic *based on security rules*.
- **{{Power over Ethernet (PoE)}}**: Transmits *electrical power over* twisted-pair cabling.
  - **{{PoE Injector}}**: Adds *power to a* non-PoE link.
  - **{{PoE Switch}}**: Switch providing *power directly to* connected devices.
  - **{{PoE Standards}}**: IEEE 802.3 specifications *defining power delivery levels*.
- **{{Hub}}**: Multi-port repeater *broadcasting data to all* ports.
- **{{Bridge}}**: Connects *two network segments* using MAC addresses.
- **{{Repeater}}**: Regenerates *signals to extend* cable distance.
- **{{Cable Modem}}**: Connects network to *internet via coaxial* cabling.
- **{{Digital Subscriber Line (DSL)}}**: High-speed *data over existing* telephone lines.
- **{{Optical Network Terminal (ONT)}}**: Converts *fiber signals to* copper Ethernet.
- **{{Network Interface Card (NIC)}}**: Hardware *enabling a device to* communicate.
  - **{{Media Access Control (MAC) Address}}**: Unique *physical identifier for* network hardware.
    - **Note:** Consists of OUI and NIC-specific serial.

### 2.6 Given a scenario, configure basic wired/wireless small office/home office (SOHO) networks

- **Internet Protocol (IP) addressing**
  - **{{IPv4}}**: 32-bit address *using dot-decimal notation* (x.x.x.x).
    - **{{Private Addresses (RFC1918)}}**: Reserved *addresses for internal* network use.
    - **{{Public Addresses}}**: Routable *addresses on the global* internet.
  - **{{IPv6}}**: 128-bit address *using hexadecimal notation* (x:x:x:x).
  - **{{Automatic Private IP Addressing (APIPA)}}**: Self-assigned *link-local address* when DHCP fails.
  - **{{Static IP}}**: Manually *configured address that does not* change.
  - **{{Dynamic IP}}**: Address *assigned automatically by* a DHCP server.
  - **{{Subnet Mask}}**: Identifies *network portion vs host* portion.
  - **{{Gateway}}**: Router providing *access to external networks*.
  - **{{Network Address Translation (NAT)}}**: Mapping *multiple private IPs to one* public IP.
  - **{{Port Forwarding}}**: Directing *external traffic to specific internal* host.
  - **{{Demilitarized Zone (DMZ)}}**: Network segment *exposed to the public* internet.
  - **{{Quality of Service (QoS)}}**: Prioritizing *critical traffic over* less important data.
  - **{{UPnP}}**: Protocol allowing *devices to discover and connect* automatically.
  - **{{MAC Filtering}}**: Allowing or blocking *devices based on physical* address.
  - **{{SSID Broadcast}}**: Announcing *wireless network name* to clients.
  - **{{Wireless Encryption}}**: Securing *wireless traffic using protocols* like WPA2/3.

### 2.7 Compare and contrast internet connection types, network types, and their characteristics

- **Internet connection types**
  - **{{Satellite}}**: Wireless *high-latency connection via space* satellites.
  - **{{Fiber}}**: Optical cabling delivering *highest speed data* via light.
  - **{{Cable}}**: Broadband internet using *television coaxial infrastructure*.
  - **{{Digital Subscriber Line (DSL)}}**: High-speed *data over existing* telephone lines.
  - **{{Cellular}}**: Wireless *mobile network data* connection (4G/5G).
  - **{{Wireless Internet Service Provider (WISP)}}**: Fixed wireless *broadband in rural areas*.
- **Network types**
  - **{{Local Area Network (LAN)}}**: Network connecting devices within *a single physical location*.
  - **{{Wide Area Network (WAN)}}**: Network connecting *geographically dispersed* locations.
  - **{{Personal Area Network (PAN)}}**: Network connecting *personal devices* around an individual.
  - **{{Metropolitan Area Network (MAN)}}**: Network spanning *a city or large campus*.
  - **{{Storage Area Network (SAN)}}**: Dedicated network *providing block-level data storage*.
  - **{{Wireless LAN (WLAN)}}**: LAN using *radio waves instead of cables*.
- **Network Characteristics**
  - **{{Bandwidth}}**: Maximum rate of *data transfer across a path*.
  - **{{Latency}}**: Time delay *for data to reach destination*.
  - **{{Throughput}}**: Actual amount of *data successfully transferred* over time.
  - **{{Jitter}}**: Variation in *packet delay* affecting real-time applications.
  - **{{Duplex}}**: Direction of *signal flow* (Half vs Full).

## 3.0 Hardware 25%

### 3.1 Compare and contrast display components and attributes

- **Types**
  - **{{Liquid Crystal Display (LCD)}}**: Display using *liquid crystals modulated by* light.
    - **{{In-Plane Switching (IPS)}}**: LCD panel type with *excellent color and viewing* angles.
    - **{{Twisted Nematic (TN)}}**: LCD panel type with *fastest response times* but poor angles.
    - **{{Vertical Alignment (VA)}}**: LCD panel type with *high contrast and deep* blacks.
  - **{{Organic Light-Emitting Diode (OLED)}}**: Display where *each pixel emits its own* light.
  - **{{Mini Light-Emitting Diode (Mini-LED)}}**: Backlight technology using *thousands of tiny LEDs* for contrast.
- **{{Touch Screen / Digitizer}}**: Layer converting *physical touch into digital* signals.
- **{{Inverter}}**: Component converting *DC power to AC* for CCFL backlights.
- **Attributes**
  - **{{Pixel Density (PPI)}}**: Measurement of *image sharpness based on resolution* and size.
  - **{{Refresh Rate}}**: Frequency at which *image is redrawn on screen* (Hz).
  - **{{Screen Resolution}}**: Total count of *horizontal and vertical pixels* (e.g., 1920x1080).
  - **{{Color Gamut}}**: Range of *colors a display can accurately* reproduce.
  - **{{Aspect Ratio}}**: Proportional relationship between *width and height of display*.
  - **{{Luminance / Brightness}}**: Measure of *luminous intensity emitted* by screen (nits).

### 3.2 Summarize basic cable types and their connectors, features, and purposes

- **Network cables**
  - **{{Copper}}**: Metal cabling *transmitting data via electrical pulses*.
    - **{{Categories}}**: Performance standards *defining data speed and bandwidth* (e.g., Cat6).
    - **{{T568A / T568B Standards}}**: Standardized *pinout patterns for RJ45 connectors*.
    - **{{Coaxial}}**: Cable with *central conductor and outer shield* for RF.
    - **{{Shielded Twisted Pair (STP)}}**: Cabling with *metallic foil to reduce interference*.
    - **{{Direct Burial}}**: Cable *rated for installation underground* without conduit.
    - **{{Unshielded Twisted Pair (UTP)}}**: Common cabling *lacking extra electrical shielding*.
  - **{{Plenum-Rated}}**: Cable *with fire-retardant jacket* for air spaces.
  - **{{Optical}}**: Cabling *transmitting data via light pulses*.
    - **{{Single-Mode}}**: Fiber used for *long distances using laser* light.
    - **{{Multimode}}**: Fiber used for *shorter distances using LED* light.
- **Peripheral cables**
  - **{{USB 2.0}}**: Legacy USB standard *supporting up to 480 Mbps*.
  - **{{USB 3.0}}**: USB standard *supporting up to 5 Gbps* (SuperSpeed).
  - **{{Serial}}**: Legacy *low-speed data transfer* one bit at a time.
  - **{{Thunderbolt}}**: High-speed *interface combining PCIe and DisplayPort*.
- **Video cables**
  - **{{High-Definition Multimedia Interface (HDMI)}}**: Digital *standard for high-definition video* and audio.
  - **{{DisplayPort}}**: Digital *standard designed for computer* displays.
  - **{{Digital Visual Interface (DVI)}}**: Digital *and analog video interface* (legacy).
  - **{{Video Graphics Array (VGA)}}**: Analog *video interface using 15 pins*.
  - **{{USB-C}}**: Reversible *connector supporting video via Alt Mode*.
- **Hard drive cables**
  - **{{Serial Advanced Technology Attachment (SATA)}}**: Interface for *connecting internal storage devices*.
  - **{{External SATA (eSATA)}}**: SATA *variation designed for external* storage.
- **{{Adapters}}**: Devices *converting one connector type* to another.
- **Connector types**
  - **{{RJ11}}**: 6-pin *connector used for telephone* lines.
  - **{{RJ45}}**: 8-pin *connector used for Ethernet* networking.
  - **{{F-Type}}**: Screw-on *connector used for coaxial* cabling.
  - **{{Straight Tip (ST)}}**: Bayonet-style *fiber connector with circular* tip.
  - **{{Subscriber Connector (SC)}}**: Square-shaped *push-pull fiber connector*.
  - **{{Lucent Connector (LC)}}**: Small *form-factor snap fiber connector*.
  - **{{Punchdown Block}}**: Mechanism *terminating individual copper wires* (e.g., 110-block).
  - **{{MicroUSB / MiniUSB}}**: Smaller *legacy connectors for mobile* devices.
  - **{{USB-C}}**: Reversible *high-speed connector for multiple* functions.
  - **{{Molex}}**: Power *connector for legacy internal* components.
  - **{{Lightning}}**: Proprietary Apple *connector for charging and data*.
  - **{{DB9}}**: D-subminiature *connector used for serial* ports.

### 3.3 Compare and contrast RAM characteristics

- **{{Random-Access Memory (RAM)}}**: Volatile storage for *currently active data*.
- **Form factors**
  - **{{Small Outline DIMM (SODIMM)}}**: Compact memory module *designed for laptops*.
  - **{{Dual In-line Memory Module (DIMM)}}**: Standard memory module *designed for desktops*.
- **{{Synchronous Dynamic RAM (SDRAM)}}**: RAM synchronized *with the system clock*.
- **Double Data Rate (DDR) iterations**
  - **{{DDR}}**: Transfers data *twice per clock cycle*.
  - **{{DDR2}}**: Twice the *speed of original DDR*.
  - **{{DDR3}}**: Lower *voltage and higher speeds* than DDR2.
  - **{{DDR4}}**: High *density and faster speeds* for modern systems.
  - **{{DDR5}}**: Latest standard *providing maximum bandwidth*.
- **{{Error-Correcting Code (ECC)}}**: RAM that *detects and fixes* memory errors.
- **{{Non-ECC}}**: Standard RAM *lacking error correction* capabilities.
- **Channel configurations**
  - **{{Single Channel}}**: Memory controller *accesses one module* at a time.
  - **{{Dual Channel}}**: Controller *accesses two modules* simultaneously.
  - **{{Triple / Quad Channel}}**: Increasing *bandwidth by using more* paths.

### 3.4 Compare and contrast storage devices

- **{{Storage Devices}}**: Components used for *persistent data retention*.
- **Hard drives**
  - **{{Hard Disk Drive (HDD)}}**: Magnetic storage using *spinning platters and heads*.
  - **{{Spindle Speeds}}**: Rotational speed *determining data access latency* (RPM).
  - **Form factors**
    - **{{2.5-inch}}**: Standard size for *laptops and small devices*.
    - **{{3.5-inch}}**: Standard size for *desktop computer storage*.
- **Solid-state drives**
  - **{{Solid-State Drive (SSD)}}**: High-speed storage using *non-volatile flash memory*.
  - **Communications interfaces**
    - **{{Non-Volatile Memory Express (NVMe)}}**: Protocol optimized for *high-speed flash storage* access.
    - **{{SATA}}**: Common interface for *connecting internal storage* devices.
    - **{{Peripheral Component Interconnect Express (PCIe)}}**: High-speed bus *supporting direct SSD* connections.
    - **{{Serial Attached SCSI (SAS)}}**: Enterprise interface for *high-performance disk* storage.
  - **Form factors**
    - **{{M.2}}**: Compact expansion *card format for SSDs*.
    - **{{Mini-SATA (mSATA)}}**: Smaller SATA *format for legacy* laptops.
- **Drive configurations**
  - **{{Redundant Array of Independent Disks (RAID)}}**: Combining multiple disks *for performance or redundancy*.
  - **{{RAID 0 (Striping)}}**: Spreads data *across disks for speed*.
  - **{{RAID 1 (Mirroring)}}**: Duplicates data *across disks for redundancy*.
  - **{{RAID 5 (Striping with Parity)}}**: Distributed parity *providing speed and fault* tolerance.
  - **{{RAID 6 (Striping with Double Parity)}}**: Allows for *simultaneous failure of two* disks.
  - **{{RAID 10 (1+0)}}**: Combination of *mirroring and striping* features.
- **Removable storage**
  - **{{Flash Drive}}**: Portable *USB-based solid-state* storage.
  - **{{Memory Card}}**: Small *flash storage used in* cameras/phones.
- **{{Optical Drive}}**: Device reading *data using laser* light (CD/DVD/BD).

### 4.1 Explain virtualization concepts

- **{{Containers}}**: Lightweight *virtualization sharing the host* OS kernel.
- **{{Hypervisor}}**: Software *creating and managing virtual* machine instances.
  - **{{Type 1 (Bare Metal)}}**: Runs *directly on the host* system hardware.
  - **{{Type 2 (Hosted)}}**: Runs *as an application on* top-level OS.

  - **{{Platform as a Service (PaaS)}}**: Provider *manages hardware and* OS tools.
